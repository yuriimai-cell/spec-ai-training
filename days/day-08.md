# Day 8 — 4/10 (金): GTM実務 + 差別化整理

## 今日のゴール

AIを使ってマーケティングコンテンツを作成し、競合との差別化ポイントを整理する。4/15-16 の展示会出展に活かせるアウトプットを目指す。

## 必須課題

### 課題1: ターゲット企業向けコンテンツドラフトを作成する

記事 / LP（ランディングページ） / メール のいずれか1点以上を作成する。

**Step 1: ターゲットを明確にする**

まず Claude または Gemini に以下を聞く:

```
自社サービス（[サービス概要を簡潔に]）のターゲット企業のペルソナを作ってください。
以下を含めてください:
- 企業規模（従業員数、売上規模）
- 業種
- 担当者の役職と課題
- 導入を検討するきっかけ
```

**Step 2: コンテンツを作成する**

ターゲットが決まったら、以下のいずれかを AI に作成させる:

**パターンA: 記事**
```
[ターゲットペルソナ] 向けの記事を書いてください。
テーマ: [ターゲットが抱える課題に関するテーマ]
文字数: 1500-2000字
構成: 問題提起 → 原因分析 → 解決策 → 事例 → まとめ
読者がアクション（問い合わせ）したくなる内容にしてください。
```

**パターンB: LP文案**
```
[サービス名] のランディングページのコピーを作ってください。
構成:
1. キャッチコピー（一行で価値を伝える）
2. サブコピー（課題への共感）
3. サービスの特徴3つ
4. 導入事例
5. CTA（Call to Action）ボタンの文言
```

**パターンC: メール**
```
[ターゲットペルソナ] 宛のアプローチメールを作ってください。
- 件名: 開封率が高くなるもの3パターン
- 本文: 200字以内で要点を伝える
- 目的: 15分のオンラインミーティングの設定
```

**Step 3: 自分で修正・改善する**

AI の出力をそのまま使わず、以下を確認・修正する:
- ターゲットに合ったトーン・表現か
- 自社の強みが正確に反映されているか
- 不自然な表現や誇張がないか

成果物: `training/day-08/content-drafts.md`

---

### 課題2: X（Twitter）投稿案を3件以上作成する

**Step 1: 投稿テーマを決める**

以下のジャンルから選ぶ:
- 自社サービスの紹介
- 業界トレンドに対する見解
- 展示会出展の告知
- AI活用のTips

**Step 2: AIに複数バリエーションを出させる**

```
X（Twitter）の投稿を作ってください。
テーマ: [選んだテーマ]
条件:
- 140字以内
- ハッシュタグ2-3個
- 読者が「いいね」「リツイート」したくなる表現
- 5パターン出してください
```

**Step 3: ベスト3を選んで磨く**

5パターンの中から3つを選び、理由とともに記録する:
- 選んだ理由（なぜこのパターンが良いか）
- 修正した点（あれば）
- 想定するターゲット層

成果物: `training/day-08/x-posts.md`

### 課題3: 競合差別化ドキュメントを作成する

**Step 1: 競合をリストアップする**

```
[自社が属する業界/市場] の主要な競合企業を5-10社リストアップしてください。
各社について以下をまとめてください:
- 企業名
- 主力サービス/製品
- 強み
- ターゲット顧客層
- 価格帯（わかる範囲で）
```

**Step 2: 差別化ポイントを整理する**

```
自社サービスの特徴は以下です:
[自社の特徴を3-5個記載]

上記の競合リストと比較して、以下を整理してください:
1. 自社が競合より優れている点（3つ以上）
2. 競合が自社より優れている点（正直に）
3. 「競合が言えないこと」= 自社だけの差別化ポイント
4. ターゲット別の推奨メッセージ
```

**Step 3: AI出力の信頼性を確認する**

AI が出力した競合情報のうち、重要な内容は公式サイト等で確認する:
- 企業名は実在するか
- サービス内容は正確か
- 価格情報は最新か

成果物: `training/day-08/competitive-differentiation.md`

## Git操作

Ghostty または Cursor のターミナル（`` Ctrl + ` ``）を開いて、リポジトリフォルダに移動します。

```bash
# 研修リポジトリに移動（クローン先がデスクトップの場合）
cd ~/Desktop/spec-ai-training
git pull origin main
git checkout -b training/day-08-gtm-differentiation

```

**ファイルの作り方（Cursor を使う）:**

Cursor のサイドバーで `training/day-08` フォルダを右クリック → 「New File」でファイルを作成する。フォルダが存在しない場合は `training` を右クリック → 「New Folder」→ `day-08` と入力。

```bash

git add training/day-08/
git status
```

以下のファイルが表示されればOK:
```

new file:   training/day-08/competitive-differentiation.md
new file:   training/day-08/daily-report.md
```

```bash

git push origin training/day-08-gtm-differentiation
```

**GitHubでPRを作成:**
1. https://github.com/あなたのユーザー名/spec-ai-training を開く
2. 「Compare & pull request」をクリック
3. **base repository** が自分のリポジトリになっていることを確認（親リポジトリが選ばれていたら変更）
4. タイトル: `Day8: GTMコンテンツ + 競合差別化ドキュメント`
