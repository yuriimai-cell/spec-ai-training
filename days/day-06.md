# Day 6 — 4/8 (水): 展示会視察 + 視察レポート作成【半日】

## 今日のゴール

展示会を観察して気づきを記録し、AIを使って視察レポートにまとめる。（午前: 視察、午後: レポート作成）

> この日は展示会視察で半日使うため、課題量は通常の半分。

---

## 必須課題

### 課題1: 展示会視察（午前）

展示会を観察し、以下の観点でメモを取る。**スマートフォンのメモアプリや写真で記録する。**

**観察ポイント:**

| 観点 | 具体的に見ること |
|------|----------------|
| 注目ブース | どの企業のブースが人を集めていたか。なぜか |
| 展示手法 | デモの見せ方、ディスプレイの工夫、体験コーナーの有無 |
| 来場者対応 | 声かけのタイミング、説明の流れ、資料の配り方 |
| 配布物 | パンフレットやノベルティの種類、もらって嬉しかったもの |
| ブース導線 | ブースへの誘導方法、ブース内の動線設計 |
| 自社への示唆 | 4/15-16 の自社展示会に参考になるポイント |

**メモの取り方のコツ:**
- 写真は積極的に撮る（後でAIに情報を整理させる素材になる）
- 気になったブースは名刺交換する、またはブース名をメモする
- 「なぜこのブースは人気なのか」を自分なりに分析しながら見る

---

### 課題2: AIで視察レポートを作成する（午後）

午前のメモと写真を素材に、AIを使ってレポートを作成する。

**Step 1: メモを整理する**

1. 午前に取ったメモをテキストにまとめる（箇条書きでOK）
2. 写真がある場合: Claude に写真をアップロードして「この展示ブースの特徴を分析して」と聞く

**Step 2: AI にレポートの骨格を作ってもらう**

Claude または Gemini に以下のように指示する:

```
展示会の視察レポートを作成してください。
以下は私が現地で取ったメモです:

[ここに午前のメモを貼り付ける]

レポートの構成:
1. 視察概要（日時、場所、全体の印象）
2. 注目した企業・プロダクト（3社以上）
3. 展示手法の工夫（参考になったポイント）
4. 来場者対応で学んだこと
5. 自社展示会（4/15-16）への示唆と提案
```

**Step 3: 自分のコメントを追記して完成させる**

AI の出力に以下を追加する:
- 自分の率直な感想
- 「自社ならこうする」というアイデア
- 疑問点や追加調査が必要な項目

## Git操作

Ghostty（またはターミナル.app）を開いて、リポジトリフォルダに移動します。

```bash
# 研修リポジトリに移動（クローン先がデスクトップの場合）
cd ~/Desktop/spec-ai-training
git pull origin main
git checkout -b training/day-06-exhibition-report

# training/day-06 は用意済み。なければ: mkdir -p training/day-06
ls training/day-06
```

**ファイルの作り方:**
- ターミナルで `open training/day-06` を実行すると Finder でフォルダが開く
- 方法1: Finder 上でテキストエディット等を使いファイルを作成し、ファイル名を `exhibition-visit-report.md` にして保存
- 方法2: ターミナルで `touch training/day-06/exhibition-visit-report.md` を実行してからエディタで開く

```bash
# (ファイルを作成・編集した後)
git add training/day-06/
git status
```

以下のファイルが表示されればOK:
```
new file:   training/day-06/exhibition-visit-report.md
new file:   training/day-06/daily-report.md
```

```bash
git commit -m "docs: add Day6 exhibition visit report"
git push origin training/day-06-exhibition-report
```

**GitHubでPRを作成:**
1. https://github.com/あなたのユーザー名/spec-ai-training を開く
2. 「Compare & pull request」をクリック
3. **base repository** が自分のリポジトリになっていることを確認（親リポジトリが選ばれていたら変更）
4. タイトル: `Day6: 展示会視察レポート`
5. テンプレートに記入 → 「Create pull request」
