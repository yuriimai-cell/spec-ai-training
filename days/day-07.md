# Day 7 — 4/9 (木): 資料作成実務 + 人間/AI役割分担

## 今日のゴール

AIを活用してビジネス資料を作成する実務スキルを身につけ、人間とAIの役割分担フレームワークを構築する。

---

## 必須課題

### 課題1: スライドアウトライン作成（AIに段階的に指示）

「AIに一度に全部作らせる」のではなく、**段階的に指示を出す**方法を学ぶ。

**Step 1: 構成を作る**

Claude または Gemini に以下のように指示する:

```
ビジネスプレゼンテーションのスライド構成を考えてください。
テーマ: [自分の業務に関連するテーマを選ぶ]
（例: 「自社サービスの新規顧客向け提案資料」「チーム向け業務改善提案」「展示会ブース企画案」）

対象読者: [誰に見せるか]
目的: [何を伝えたいか / どんなアクションを促したいか]
スライド枚数: 10-15枚程度

まず構成案（スライドタイトルの一覧）だけ出してください。
```

**Step 2: 各スライドの要点を作る**

Step 1 の出力を確認し、構成に満足したら:

```
この構成でOKです。次に、各スライドに記載すべき要点を箇条書きで出してください。
各スライドにつき3-5個の要点とします。
```

**Step 3: 詳細を作る**

```
スライド3「[タイトル]」の内容を詳しく書いてください。
具体的な数値や事例を含めてください。
```

このように1スライドずつ掘り下げていく。

**なぜ段階的に指示するのか:**
- 一度に全部指示すると、AI は浅い回答になりがち
- 段階的に進めると、各ステップで方向修正できる
- 「構成OK → 要点OK → 詳細」の順番で進めると品質が上がる

成果物: `training/day-07/slide-outline.md`

### 課題2: 人間/AI役割分担マトリクスを作成する

自分の業務（GTM: セールス/マーケティング）の各工程について、「人間」「AI」「協働」のどれが最適かを整理する。

**Step 1: 業務工程のリストアップ**

Claude に以下のように聞く:

```
GTM（Go-To-Market）業務の工程を網羅的にリストアップしてください。
以下のカテゴリを含めてください:
- マーケティング（リサーチ、コンテンツ作成、SNS運用、広告）
- セールス（リード獲得、商談準備、提案書作成、フォローアップ）
- カスタマーサクセス（オンボーディング、サポート、アップセル）
```

**Step 2: 各工程に役割を割り当てる**

以下の判断基準で分類する:

| 分類 | 判断基準 | 例 |
|------|---------|-----|
| **AIが担当** | 定型的、大量処理、データ集約 | データ分析、初稿作成、定型メール |
| **人間が判断** | 創造性、共感、最終責任 | 戦略決定、重要商談、クレーム対応 |
| **協働** | AIが下書き → 人間が仕上げ | 提案書、プレゼン、コンテンツ |

**マトリクスの形式:**

```markdown
| 業務工程 | 担当 | AI活用方法 | 人間の役割 | 効率化の見込み |
|---------|------|-----------|-----------|-------------|
| 市場リサーチ | 協働 | Deep Researchで情報収集 | 情報の精査、解釈 | 工数50%削減 |
| 提案書作成 | 協働 | 構成・初稿をAI生成 | 顧客固有情報の追加、最終調整 | 工数40%削減 |
| 商談（対面） | 人間 | — | 関係構築、ニーズ把握 | — |
| ...
```

成果物: `training/day-07/role-split-matrix.md`

### 課題3: 実際の資料ドラフトを1点作成する

課題1で作ったアウトラインをもとに、実際の資料を1点作成する。

**形式は以下のいずれか:**
- Google Slides（プレゼンテーション）
- Google Docs（提案書・レポート）
- Markdown（このリポジトリに格納）

**手順:**

1. 課題1のアウトラインをもとに、AIに本文を生成させる
2. 自分の知識・経験を加えて修正する
3. デザイン・レイアウトを整える（Google Slides の場合）
4. 完成した資料のリンクまたはファイルを保存する

成果物: `training/day-07/draft-document.md`（またはGoogle Slides/Docsのリンクを記載）

## Git操作

Ghostty（またはターミナル.app）を開いて、リポジトリフォルダに移動します。

```bash
# 研修リポジトリに移動（クローン先がデスクトップの場合）
cd ~/Desktop/spec-ai-training
git pull origin main
git checkout -b training/day-07-docs-role-split

# training/day-07 は用意済み。なければ: mkdir -p training/day-07
ls training/day-07
```

**ファイルの作り方:**
- ターミナルで `open training/day-07` を実行すると Finder でフォルダが開く
- 方法1: Finder 上でテキストエディット等を使いファイルを作成し、ファイル名を `slide-outline.md` にして保存
- 方法2: ターミナルで `touch training/day-07/slide-outline.md` を実行してからエディタで開く

```bash
# (ファイルを作成・編集した後)
git add training/day-07/
git status
```

以下のファイルが表示されればOK:
```
new file:   training/day-07/slide-outline.md
new file:   training/day-07/role-split-matrix.md
new file:   training/day-07/draft-document.md
new file:   training/day-07/daily-report.md
```

```bash
git commit -m "docs: add Day7 slide outline and role-split matrix"
git push origin training/day-07-docs-role-split
```

**GitHubでPRを作成:**
1. https://github.com/あなたのユーザー名/spec-ai-training を開く
2. 「Compare & pull request」をクリック
3. **base repository** が自分のリポジトリになっていることを確認（親リポジトリが選ばれていたら変更）
4. タイトル: `Day7: 資料作成実務 + 人間/AI役割分担マトリクス`
5. テンプレートに記入 → 「Create pull request」
