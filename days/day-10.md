# Day 10 — 4/14 (火): Project A 設計

> **前提**: Cursor、Vercel、V0 のアカウントが必要です。まだの場合は `account-setup.md` の「Phase 3 開始前に準備」を先に完了してください。

## 必須課題

### 課題1: 要件定義書を作成する

自分の業務で実際に使えるWebツールの要件を定義する。

**テーマの選び方（自由。以下は例）:**
- 商談管理ダッシュボード
- 展示会リード管理ツール
- コンテンツ作成チェックリスト
- 競合比較表の自動生成ツール
- 社内FAQ検索ツール
- 議事録テンプレート生成ツール

**Step 1: アイデアをAIと壁打ちする**

Claude に以下のように相談する:

```
私はGTM（セールス/マーケティング）の仕事をしています。
日々の業務で「こんなツールがあったら便利なのに」と思うものを作りたいです。

以下の制約の中で、実現可能なWebツールのアイデアを5つ提案してください:
- 1-2ページ程度のシンプルなWebアプリ
- 非エンジニアがAIの助けを借りて1日で作れる規模
- 自分の業務で実際に使えるもの
- データベースは不要（ブラウザ上で完結）
```

**Step 2: テーマを1つ選び、要件定義書を作成する**

```
[選んだテーマ] の要件定義書を作成してください。
以下の構成で:

1. 背景（なぜこのツールが必要か）
2. ゴール（このツールで何を実現するか）
3. スコープ（作るもの / 作らないもの）
4. ターゲットユーザー（誰が使うか）
5. 主要機能（箇条書きで3-5個）
6. 画面構成（何ページ必要か、各ページの概要）
7. 非機能要件（スマホ対応の要否、パフォーマンス等）
```

**Step 3: 自分の業務知識を加えて修正する**

AI の出力を確認し、以下を修正・追記する:
- 実際の業務フローに合っているか
- 機能の優先順位は正しいか
- 「あると嬉しい機能」と「必須の機能」を区別する

成果物: `training/day-10/project-a-requirements.md`

### 課題2: V0 / Google Stitch でモックアップを作成する

モックアップ = 完成イメージの見本。実際に動くものではなく「こういう画面にしたい」というデザイン案。

**パターンA: V0 を使う（推奨）**

1. https://v0.dev を開く
2. Vercel アカウントでログイン（GitHub アカウント連携）
3. テキスト入力欄に、ツールのイメージを文章で入力する:

```
以下のWebアプリのUIを作ってください:

[要件定義書の「画面構成」と「主要機能」を貼り付ける]

デザイン要件:
- モダンでシンプルなデザイン
- レスポンシブ対応（PCとスマホ両方で見やすい）
- 日本語のUIテキスト
```

4. V0 がUIデザインとReactコードを生成する
5. プレビューで確認し、修正指示を追加で入力する（例: 「ボタンをもっと大きく」「色をブルー系に」）
6. 気に入ったデザインをスクリーンショットで保存する

**パターンB: Google Stitch を使う**

1. https://stitch.google.com を開く
2. Google アカウントでログイン
3. テキストでUIのイメージを入力する
4. 生成されたモックアップを確認・調整する

**パターンC: 手書き + AI でモックアップを作る**

1. 紙にUIの手書きスケッチを描く
2. スマホで写真を撮る
3. Claude に写真をアップロードして「このスケッチをもとにUIの詳細を説明して」と指示
4. 説明をもとに V0 や Stitch でモックアップを生成する

成果物: `training/day-10/project-a-mock.md`（モックアップのスクリーンショットまたは説明を記載）

### 課題3: Vercel 初期セットアップ

Day11 でデプロイ（公開）するために、Vercel のセットアップを完了させる。

**手順:**

1. https://vercel.com を開く
2. 「Continue with GitHub」をクリック
3. GitHub アカウントでログインする
4. アカウント作成画面が表示されたら:
   - 名前を入力
   - 「Hobby」プラン（無料）を選択
   - 「Continue」をクリック
5. ダッシュボードが表示されれば成功

**この画面が出たらスクリーンショットを撮ってください（Vercel ダッシュボード）。**

**チーム招待メールがある場合:**
- メールの「Accept Invitation」をクリック
- チームに参加する

**動作確認（まだデプロイはしない）:**
- ダッシュボードが表示されること
- 「New Project」ボタンが押せること（押すだけ。まだ作成しない）

成果物: セットアップ完了の確認（日報に記載）

## Git操作

Ghostty または Cursor のターミナル（`` Ctrl + ` ``）を開いて、リポジトリフォルダに移動します。

```bash
# 研修リポジトリに移動（クローン先がデスクトップの場合）
cd ~/Desktop/spec-ai-training
git pull origin main
git checkout -b training/day-10-project-a-design

```

**ファイルの作り方（Cursor を使う）:**

Cursor のサイドバーで `training/day-10` フォルダを右クリック → 「New File」でファイルを作成する。フォルダが存在しない場合は `training` を右クリック → 「New Folder」→ `day-10` と入力。

```bash
# (ファイルを作成・編集した後)
git add training/day-10/
git status
```

以下のファイルが表示されればOK:
```
new file:   training/day-10/project-a-requirements.md
new file:   training/day-10/project-a-mock.md
new file:   training/day-10/daily-report.md
```

```bash
git commit -m "docs: add Day10 Project A requirements and mockup"
git push origin training/day-10-project-a-design
```

**GitHubでPRを作成:**
1. https://github.com/あなたのユーザー名/spec-ai-training を開く
2. 「Compare & pull request」をクリック
3. **base repository** が自分のリポジトリになっていることを確認（親リポジトリが選ばれていたら変更）
4. タイトル: `Day10: Project A 要件定義 + モックアップ`
5. テンプレートに記入 → 「Create pull request」
