# 学習インフラ

最終更新: 2026-04-04

> この文書は研修の運用ルールを定義する。不明点があればまずここを確認すること。

---

## 1) Phase別の提出ルール

### Phase 1（Day1-Day3）: オリエンテーション期間

- Git/GitHub研修前のため、PR提出は不要
- 成果物は Google Docs / Google Slides で作成し、指定の Google Drive フォルダに格納する
- 格納後、Slack の研修チャンネルに共有リンクを投稿して完了報告する
- Day4のGitHub基礎完了後、過去分もリポジトリにPushすることを推奨

### Phase 2-3（Day4-Day13）: 日次実行ループ

Day5以降、毎日以下のループを完了させる。

1. `training/day-xx/` に成果物を作成
2. ブランチをPush（ブランチ名: `training/day-xx-<topic>`）
3. PR作成（テンプレート: `templates/day-pr-body-template.md`）
4. Slack通知を確認
5. レビューコメントを反映
6. Merge

これを完了必須ループとする。

レビュア応答が遅れる場合の暫定完了条件:
- PR作成済み
- Slack通知確認済み
- クイズ100点達成

Merge後に最終完了へ更新する。

## 2) クイズループ

1. `resources.md` の該当Day教材を読む
2. `quizzes/index.html` をブラウザで開いてクイズを受験する
3. 100点になるまで再受験する（当日内は無制限）
4. 100点の合格画面をスクリーンショットし、PR本文に添付する
5. クイズ合格後にMerge可能

## 3) 非同期レビュー運用ルール

- 対面での説明待ちで進行を止めない。次のDayに進んでよい
- 不明点はドキュメント、`resources.md` の参照リンク、PRコメントで解決する
- レビューはGitHub上で非同期に実施する
- レビューコメントへの対応は翌営業日中に完了させる

## 4) 成果物の品質基準

提出物は以下の基準を満たすこと。

- 課題の目的に対して回答になっている
- 第三者が読んで理解できる構成になっている
- AIの出力をそのまま貼り付けるのではなく、自分の判断・編集が加わっている
- ソースや根拠がある場合は引用している

## 5) PR本文テンプレート（最低限の記載事項）

テンプレートファイル: `templates/day-pr-body-template.md`

最低限の記載事項:
- 当日の目的
- 成果物パス
- クイズ証跡（100点）
- 参照した教材リンク
- レビューしてほしい観点

## 6) ローカル限定コンテキスト方針

- 公開リポジトリに社内機密情報を含めない
- 内部コンテキストが必要な場合は以下に保存する
  - `references/farleap-context.local.md`（gitignore対象）
- テンプレート: `references/farleap-context.example.md`
